Git合并冲突问题

2018-11-30

今日小结【解决合并冲突问题】

情况描述：这几天本地做的总结没有及时上传至github，然后呢今天把github上的文件改了一下。
同时想先把远程修改的github上的readme文件git pull下本地，同步一下在继续工作。

首次git pull

报错【磨人的小妖精】：

大概意思是我在本地改了readme.md文件，导致远程和本地的文件不一致，无法拉取（pull）下来，要我先把本地的先commit，再git pull，就会将在github上更改的文字合并到本地，

再次git pull

报错【磨人的小妖精】：

大概意思是readmetest.md文件存在冲突，必须手动解决冲突后再执行git add，git commit，git push之类的命令提交

参考廖雪峰-创建合并分支 ，解决冲突

小结
当Git无法自动合并分支时，就必须首先解决冲突。解决冲突后，再提交，合并完成。
解决冲突就是把Git合并失败的文件手动编辑为我们希望的内容，再提交。

分支合并问题：

情况一：顺利合并分支
一般都是切换到master然后再执行git merge <name>

关于分支
Git鼓励大量使用分支：常用命令

*查看分支：git branch

*创建分支：git branch

*切换分支：git checkout

*创建+切换分支：git checkout -b

*合并某分支到当前分支：git merge

*删除分支：git branch -d
