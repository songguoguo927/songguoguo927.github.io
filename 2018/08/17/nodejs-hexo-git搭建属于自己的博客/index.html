nodejs+hexo+git搭建属于自己的博客

2018-08-17

大一下的时候，学长学姐就开始建议我们搭博客，但当时自己真的太渣，尝试搭过，但一路上总是那么多坎坎坷坷，电脑不是这有问题就是那有问题，加上遇到困难就畏缩害怕，问了学长学姐后，自己还是没有搭起来，便放弃了。还理所当然觉得还为之过早，现在觉得那种想法可以理解但还是不太正确。
也并不是说现在有多厉害，而是随着经历的多了，慢慢的学会不退缩，学会能独立解决就解决，不害怕困难，能平下心来吧。
前几天在掘金上看了一篇文章《2018，你该搭建自己的博客了！》看的蛮有感触的，所以决定重拾起当时放下的任务，其实在此之前的一段时间，搭博客这件事在我心里就是一件不敢面对的事，虽然大家都说简单，但我脑中已给他定下难这个印象了。所以一直没有去实现。
看了这篇文章后我花了一个下午搭好了，并没有像文章中那样顺顺利利，也遇到过几个问题，但都通过自己谷歌解决了。
说实话，真的很开心，就像拔掉了心中的刺。

现在废话不多说，我们搭博客。按照http://ryane.top 这个人的博客文章《2018，你该搭建自己的博客了！》。
我一步步做下来，有以下感悟：

第一，认识Github Page

之前对git有过了解，也熟悉一些基本命令，写过一篇关于如何将本地文件上传至github上，所以关于git这一块就不用说了，已经安装好了git，也注册了，相关SSH也配置好了。
所以着重认识什么是 Github Page，它可以被认为是用户编写的、托管在github上的静态网页 。
它可以绑定我们的一个域名，简单快捷，为我们提供一个免费的服务器，免去了自己搭建服务器和写数据库的麻烦。

第二，安装Node.js, npm

去官网下载就好，根据电脑64 or32位自行下载安装（直接无脑下一步）对应版本，暂时不用配置环境变量。
由于新版的NodeJS已经集成了npm，所以之前npm也一并安装好了.

判断git，nodejs,npm是否安装成功，直接在终端cmd，git --version,node -v都显示对应的版本号说明安装成功.

在这里，命途多舛一：安装时，npm报错，然后我测试npm -v发现没有出现对应的版本，再静下心来看看一大堆英文说啥：原来是文件夹权限问题

解决办法：在nodejs文件夹右键属性》安全》编辑，把方框里的权限改成允许。再确定》确定。

第三，Hexo（博客框架）安装

桌面右键鼠标，点击Git Bash Here，输入npm命令即可安装

npm install hexo-cli -g或 $ npm install -g hexo

npm install hexo-deployer-git --save

第一条命令后，出现如下，说明安装完成。

在这里，命途多舛二：第二条命令时出现报错，提示 hexo不是内部命令问题。
参考文章：https://blog.csdn.net/qq_27093465/article/details/72954725
原来是 高级环境变量问题
解决办法：在系统环境变量path加上C:\Program Files\nodejs\node_global\node_modules\hexo\node_modules.bin（具体路径结合自己的安装路径情况分析）

再判断一下问题是否解决，直接在终端cmd，hexo -v or hexo分别出现版本提示，或一些hexo基本命令
如下图：

第二条命令可以现在安装，也可以在后面安装，都可以，反正躲不过。

第四，Hexo初始化配置

然后在你新建的 （想在哪建就在哪件，自己记清楚就行）一个文件夹(空的)下右键git bash here

hexo init



这样就完成啦。
我们再执行以下命令：

hexo generate
hexo server

登录localhost:4000，既可以看到本地效果

第五，将博客部署到Github Pages上

我看的那篇文章上更详细。

以后我们每写完一篇博客想部署到Github Pages上，只要执行以下两条命令即可。

$ hexo g 生成文件
$hexo d #部署 d是deploy的简写

也可以将上面的命令合并为
hexo g -d

好啦就到这啦，也没那么难嘛。我们都可以的。
