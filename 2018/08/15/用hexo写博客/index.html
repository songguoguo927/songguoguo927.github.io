学会如何用hexo写自己的博客

（文章摘要）在博客写文章

一、用hexo发表新文章

$ hexo n "文章标题" n即new的简写

在Hexo右键gitbash here，执行该条命令后，会在项目\Hexo\source_posts中生成文章标题.md文件，用编辑器打开编写即可。
或者自己直接在\Hexo\source_posts中新建一个md文件。
编写时，语法格式遵循markdown 不用多了解，会用就行，具体参照https://blog.csdn.net/csdn1027192411/article/details/48876739

二，本地查看博客效果

写完后，觉得不错可以依次执行以下命令：

$hexo clean #清除缓存文件 为了避免不必要的错误，建议在生成静态文件前执行
$hexo generate #生成静态文件 也可简写为hexo g
$hexo server 也可简写为hexo s

执行到这你就可以启动Hexo服务器在本地查看啦，在浏览器打开 http://localhost:4000/ 即可查看文章效果了

ps.每次hexo s后只要不停止，在本地修改文章保存，在localhost:4000网页刷新即可随时看到更新的文章，而不用每次修改每次生成和启动服务才能查看本地效果。

三，如果想将博客部署到github上

执行下面命令：

$ hexo g #生成文件
$hexo d #部署 d是deploy的简写

也可以将上面的命令合并为
hexo d -g或 hexo g -d

执行完后，这样我们自己的博客在Github Pages服务器上就部署好了，浏览器输入https://songguoguo927.github.io/ （这是我当时设置好的我的，你应该用你自己的）就可以看到了与本地相同的效果啦
这个是当时在搭建阶段时 将本地的Hexo文件更新到Github的库中 那一步中相关联的。
