Git合并冲突问题

2018-11-30

今日小结【解决合并冲突问题】

情况描述：这几天本地做的总结没有及时上传至github，然后呢今天把github上的文件改了一下。
同时想先把远程修改的github上的readme文件git pull下本地，同步一下在继续工作。

首次git pull

报错【磨人的小妖精】：

大概意思是我在本地改了readme.md文件，导致远程和本地的文件不一致，无法拉取（pull）下来，要我先把本地的先commit，再git pull，就会将在github上更改的文字合并到本地，

再次git pull

报错【磨人的小妖精】：

大概意思是readmetest.md文件存在冲突，必须手动解决冲突后再执行git add，git commit，git push之类的命令提交

参考廖雪峰-创建合并分支 ，解决冲突

小结
当Git无法自动合并分支时，就必须首先解决冲突。解决冲突后，再提交，合并完成。
解决冲突就是把Git合并失败的文件手动编辑为我们希望的内容，再提交。

分支合并问题：

情况一：顺利合并分支
一般都是切换到master然后再执行git merge <name>

关于分支
Git鼓励大量使用分支：常用命令

*查看分支：git branch

*创建分支：git branch

*切换分支：git checkout

*创建+切换分支：git checkout -b

*合并某分支到当前分支：git merge

*删除分支：git branch -d

LAMP搭建discuz

2018-10-26

本身电脑系统为windows64位
前提：

一,装好L

  1. (L:Linux)在电脑上下载了VMware Ubuntu64位镜像 搭建虚拟机上的linux系统.首先将系统本身自带的配置文件/etc/apt/sources.list 进行修改。文件/etc/apt/sources.list是一个普通可编辑的文本文件，保存了ubuntu软件更新的源服务器的地址。把dep那一行的archive.ubuntu.com和security那个都改成mirrors.ailiyun.com
    【1】更新所有包
    sudo apt update && sudo apt upgrade
    【2】安装好所需软件包
    语法：sudo apt install [安装的名字]
    安装的名字可为多个，空格隔开

二,装好AMP

A:apache2
M:Mariadb
P:php
控制台输入：sudo apt install apache2 mariadb-server php php-mysql libapache2-mod-php php-xml

三,逐个进行测试是否安装成功

（1）测试apache安装是否成功
在虚拟机的Linux系统自带的firefox（火狐）浏览器中输入
127.0.0.1(或者localhost)
出现以下界面则apache安装成功

（2）测试mysql安装是否成功
sudo myql_secure_installation
(输到sec记得使用tab键，多多使用防出错）
按照他的操作后
出现Thanks for using MariaDB！
下一步
sudo -i
mysql -u root -p
出现MariaDB[(none)]>
则表明安装成功
（3）测试php安装是否成功（测试Apache能否解析PHP）
接着上面

先
exit
再输入
~# cd /var/www/html

~#ls
查看一下该目录里有啥（可不看）
在这目录下创建一个文件test.php
可用vim or nano，我这里用nano，当然你可以用vim（用法具体自行百度）

#nano test.php
(该语法的含义表示用nano创建一个文件，如果目录下本身没有该文件，则创建一个空白的，有则打开已有的）
在nano编译环境中输入如下内容
<?php phpinfo(); ?>
保存退出
再在浏览器输入127.0.0.1/test.php
出现如下图所示则表明安装成功

四,discuz的下载与安装

js学习-函数详细讲解

2018-08-22

前情提要：

数据类型
基本数据类型
number，string，null，boolean
引用数据类型
object，function

区别:引用类型操作内存地址

正文：

函数详细讲解

基础知识

function
1.“洗衣机”，具备某个功能的一个方法。方法本身无意义，执行了才有价值。
2.两部分组成
创建: function 方法名(){//函数体:存放的是某个功能实现的js代码}
1.首先开辟一个新的内存空间，浏览器为其分配一个16进制的内存空间eg，xxxfff000，
2.把函数体里的js代码当成字符串存进内存空间
3.再把内存空间的地址名赋给当前函数名
执行:方法名();创建一个函数可多次执行，且互相执行之间没有关系。
1.sum；表示当前函数本身；sum();表示执行当前函数，函数执行的目的:把函数之前存储的那些字符串变为js代码，然后让js代码从上之下依次执行。
2.函数在执行时，形成一个私有作用域供函数体中代码执行
3.把函数体中的代码字符串变为代码从上到下执行
3.实例:两个数（固定）求和的函数
null 挖坑啥没干（推荐）， 0坑都没挖。
4.闭包:函数执行时形成的私有作用域把我们函数体中的代码都包起来了，保护了里面的私有变量不受外界干扰（外面获取不到，修改不了），我们把函数执行时形成的这种保护机制叫做闭包。
形参（函数暴露的入口）
实例，求任意两个数之和。sum(num1，num2);
return（当前函数提供的出口）
1,函数体中没有return，由于闭包机制在方法外拿不到里面的total，默认返回值undefined。
2, 有return，return total;返回给外面的是total变量存储的值，不是变量total。要想知道具体的值多少，还得在外边var total=sum(100,200);定义一个变量来接收返回值，此次total不是里面的total，之所以可以重名是因为里面的由于闭包机制传不到外面。所以外面可以这样命名。
arguments
1.函数天生自带的（不管写没写形参，也不管是否传值，arguements始终存在）用来接收函数传递进来的参数值的一个集合（传几个存几个）。
2.是一个object数据类型的值，以数字作为索引从0开始，类数组，有length。
3.实例:“任意数求和”:给我任意个数，求出它们的和。
关键点:具体几个数不知道，没办法定义形参的个数。利用arguments.length循环类数组集合。
优化1:把传进来的值转换为number类型，防止出现字符串拼接
优化2:对非有效数字不累加，防止结果为NaN（isNaN（）判断）
匿名函数:没有起名字的函数(-相反有函数名字的叫实名函数)
js中常用两种方式
1.自执行函数（也叫立即执行函数）:定义执行一起
;（function（num）{}）（100）;//第一个分号作用为了防止它上面的代码结束不加。分号还可以用+，-，！，~替换（装×用）。
2.函数表达式:把函数定义的部分当成一个值赋值给一个变量或者元素的某一个行为
oDiv.onclick=function(){}

nodejs+hexo+git搭建属于自己的博客

2018-08-17

大一下的时候，学长学姐就开始建议我们搭博客，但当时自己真的太渣，尝试搭过，但一路上总是那么多坎坎坷坷，电脑不是这有问题就是那有问题，加上遇到困难就畏缩害怕，问了学长学姐后，自己还是没有搭起来，便放弃了。还理所当然觉得还为之过早，现在觉得那种想法可以理解但还是不太正确。
也并不是说现在有多厉害，而是随着经历的多了，慢慢的学会不退缩，学会能独立解决就解决，不害怕困难，能平下心来吧。
前几天在掘金上看了一篇文章《2018，你该搭建自己的博客了！》看的蛮有感触的，所以决定重拾起当时放下的任务，其实在此之前的一段时间，搭博客这件事在我心里就是一件不敢面对的事，虽然大家都说简单，但我脑中已给他定下难这个印象了。所以一直没有去实现。
看了这篇文章后我花了一个下午搭好了，并没有像文章中那样顺顺利利，也遇到过几个问题，但都通过自己谷歌解决了。
说实话，真的很开心，就像拔掉了心中的刺。

现在废话不多说，我们搭博客。按照http://ryane.top 这个人的博客文章《2018，你该搭建自己的博客了！》。
我一步步做下来，有以下感悟：

第一，认识Github Page

之前对git有过了解，也熟悉一些基本命令，写过一篇关于如何将本地文件上传至github上，所以关于git这一块就不用说了，已经安装好了git，也注册了，相关SSH也配置好了。
所以着重认识什么是 Github Page，它可以被认为是用户编写的、托管在github上的静态网页 。
它可以绑定我们的一个域名，简单快捷，为我们提供一个免费的服务器，免去了自己搭建服务器和写数据库的麻烦。

第二，安装Node.js, npm

去官网下载就好，根据电脑64 or32位自行下载安装（直接无脑下一步）对应版本，暂时不用配置环境变量。
由于新版的NodeJS已经集成了npm，所以之前npm也一并安装好了.

判断git，nodejs,npm是否安装成功，直接在终端cmd，git --version,node -v都显示对应的版本号说明安装成功.

在这里，命途多舛一：安装时，npm报错，然后我测试npm -v发现没有出现对应的版本，再静下心来看看一大堆英文说啥：原来是文件夹权限问题

解决办法：在nodejs文件夹右键属性》安全》编辑，把方框里的权限改成允许。再确定》确定。

第三，Hexo（博客框架）安装

桌面右键鼠标，点击Git Bash Here，输入npm命令即可安装

npm install hexo-cli -g或 $ npm install -g hexo

npm install hexo-deployer-git --save

第一条命令后，出现如下，说明安装完成。

在这里，命途多舛二：第二条命令时出现报错，提示 hexo不是内部命令问题。
参考文章：https://blog.csdn.net/qq_27093465/article/details/72954725
原来是 高级环境变量问题
解决办法：在系统环境变量path加上C:\Program Files\nodejs\node_global\node_modules\hexo\node_modules.bin（具体路径结合自己的安装路径情况分析）

再判断一下问题是否解决，直接在终端cmd，hexo -v or hexo分别出现版本提示，或一些hexo基本命令
如下图：

第二条命令可以现在安装，也可以在后面安装，都可以，反正躲不过。

第四，Hexo初始化配置

然后在你新建的 （想在哪建就在哪件，自己记清楚就行）一个文件夹(空的)下右键git bash here

hexo init



这样就完成啦。
我们再执行以下命令：

hexo generate
hexo server

登录localhost:4000，既可以看到本地效果

第五，将博客部署到Github Pages上

我看的那篇文章上更详细。

以后我们每写完一篇博客想部署到Github Pages上，只要执行以下两条命令即可。

$ hexo g 生成文件
$hexo d #部署 d是deploy的简写

也可以将上面的命令合并为
hexo g -d

好啦就到这啦，也没那么难嘛。我们都可以的。
